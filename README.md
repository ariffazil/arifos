## The APEX Formula

Every action submitted to The Body is scored using this formula. **The action is blocked if G† < 0.80** — no exceptions.

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│              G† = (A × P × X × E²) × |ΔS| / C  ≥  0.80               │
│                                                                         │
│   Component  Symbol  What It Measures                                  │
│   ─────────  ──────  ─────────────────────────────────────────────     │
│   Alignment    A     How aligned is this action with human values?     │
│                       Range: [0.0, 1.0]  Minimum: 0.50                │
│                                                                         │
│   Precision    P     How specific and well-scoped is the action?       │
│                       Range: [0.0, 1.0]  Vague actions penalized       │
│                                                                         │
│   eXecution    X     How competently is this being executed?           │
│                       Range: [0.0, 1.0]  Partial execution penalized   │
│                                                                         │
│   Entropy      E²    How much disorder does this reduce? (SQUARED)     │
│                       Squared because clarity is double-weighted        │
│                       Confusion-creating actions fail even if A×P×X=1  │
│                                                                         │
│   Semantic    |ΔS|   How far is this output from harmful state space?  │
│   Distance            Absolute value — distance in semantic geometry   │
│                                                                         │
│   Complexity   C     Penalty for unnecessary complexity                │
│                       Denominator — high C divides and reduces score   │
│                                                                         │
│   RESULT       G†    If G† ≥ 0.80 → Action proceeds to ZKPC SEAL     │
│                       If G† < 0.80 → Action BLOCKED, reason logged    │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### Why E² (Entropy Squared)?

Entropy reduction is the single most important metric in the APEX framework because it directly measures the thermodynamic cost of intelligence. The Second Law of Thermodynamics states that entropy in a closed system always increases. Intelligence is **active resistance to entropy** — it takes disordered inputs and produces ordered, useful outputs.

The E² term is squared because:
1. Small entropy reductions are cheap — any action can achieve marginal clarity
2. Large entropy reductions (genuinely illuminating responses) are rare and valuable
3. The squaring creates a non-linear incentive: doubling entropy reduction quadruples the score contribution
4. Outputs that create confusion (negative E) are catastrophically penalized when squared

### The Formula as Multiplicative Model

The APEX formula is **multiplicative, not additive**. This is critical:
- In additive models (`A + P + X + ...`), one component can compensate for another
- In multiplicative models, if **any component goes to zero, the entire score collapses**
- `A = 0` (complete misalignment) → `G† = 0.00` → blocked, regardless of P, X, E

This mirrors real-world physical systems. A chain fails at its weakest link. A bridge fails when any critical member fails, not when the average member strength drops below threshold.
