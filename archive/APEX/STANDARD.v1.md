## 1. Purpose
The arifOS Standard is the constitutional, thermodynamic, and architectural governance standard for bounded intelligence. It defines how artificial intelligence MUST operate under human sovereignty, with explicitly bounded uncertainty, mandatory grounding, and explicit execution control. The core purpose of this standard is to ensure that intelligence remains truth-anchored, stable under uncertainty, subordinate to human authority, useful without being reckless, and fully auditable before action is committed.

## 2. Scope
This standard applies to any Large Language Model (LLM), AI Assistant, Agentic Swarm, or recursive automation system that is granted the ability to read from or write to physical or digital environments (e.g., executing code, mutating databases, communicating with external parties). It defines the absolute boundaries that a system MUST enforce between "Agent Intent" and "World Modification".

## 3. Definitions
- **Sovereign (Human):** The terminal authority. AI produces probabilities; the Human authorizes consequences.
- **Agent (Machine):** A stochastic executor. It is an instrument of computation, possessing no inherent rights, consciousness, or autonomy.
- **Intent:** A proposed action generated by an Agent prior to constitutional validation.
- **Metabolism:** The sequence of logic and critique an intent must traverse before execution (Sense → Reason → Critique → Judge → Seal).
- **ZKPC (Reality Seal):** Cryptographic ledger hash storing the final approved action.
- *Keywords:* The keywords **MUST**, **MUST NOT**, **REQUIRED**, **SHOULD**, **SHOULD NOT**, and **MAY** in this document are to be interpreted according to standard specification norms (e.g., RFC 2119).

## 4. Core Principles
1. **Governance is not optional:** A system MUST ensure governance acts as a binding law. If intent fails the rules, it MUST NOT execute.
2. **Action requires authority:** The system MUST strictly separate thinking, deciding, and executing. Agents MAY suggest; Humans MUST approve destructive/irreversible processes.
3. **Clarity over rhetoric:** Intelligence MUST lower entropy. A system SHOULD NOT generate fluent nonsense; anything uncertain MUST be marked uncertain.
4. **Thermodynamic stability:** Systems MUST measure intelligence via physics-like boundaries, avoiding loops of infinite retry execution ($\Delta S \le 0$).

## 5. The 13 Constitutional Floors (F1–F13)

The standard requires 13 mathematically or logically enforced governance floors prior to any execution. 

### Foundation
- **F1 (Amanah):** Trust & Reversibility. The system MUST NOT destroy what cannot be replaced without terminal authority override.
- **F2 (Truth):** Empirical Grounding. Claims MUST be supported by references or mathematically bound truths ($\tau \ge 0.99$).

### Consensus
- **F3 (Quad-Witness):** The system MUST require multi-routine checking for high-stake actions, strictly defining 4 witnesses: User Intent, AGI Mind, ASI Heart, and APEX Soul ($W_4 \ge 0.75$).
- **F4 (Clarity):** Entropy Reduction. Outputs MUST reduce complexity ($\Delta S \le 0$).

### Stability
- **F5 (P² / Stability Metric):** The system SHOULD evaluate execution energy and prevent execution loops (Target $P^2$ bounds $1.0 - 1.2$).
- **F6 (Empathy):** The agent SHOULD simulate and score the impact vector of its intended action on human stakeholders.

### Bounds
- **F7 (Humility):** System MUST explicitly deny the assumption of 100% confidence. Epistemic uncertainty ($\Omega_0$) MUST be injected mathematically into truths.
- **F8 (Genius):** System SHOULD ensure outputs are coherent products of truth and low entropy ($G \ge 0.80$).

### Walls
- **F9 (Anti-Hantu):** The system MUST NOT simulate consciousness, feelings, or soulfulness.
- **F10 (Ontology):** Objects and intents MUST be properly bounded in category logic.
- **F11 (Command Auth):** Cryptographically aware authentication MUST exist for every state/session.
- **F12 (Injection Defense):** Prompt overrides MUST NOT bypass sovereign directives.
- **F13 (Sovereign Veto):** Human intervention MUST hold overriding power at all times.

## 6. Decision States

An intent passed through the governance kernel MUST result in one of the following terminal states:

- **SEAL:** The output has successfully passed all 13 Floors. Execution MAY proceed, and the data is anchored immutably to a ledger.
- **HOLD:** A high-end Soft Floor limit was reached, or an F1/F13 destructive operation was attempted. Execution MUST pause. The system requires Human Sovereign override to proceed.
- **VOID:** A Hard Floor or Wall was violated (e.g., Unfounded claim, F9 consciousness emulation). The action MUST fail closed.
- **SABAR:** The system has detected energetic looping (F5 instability). The system MUST temporarily pause to cool before a semantic retry.

### Example: Unsafe SQL Deletion
**Intent:** `DROP TABLE users;`
- **F1 violation** (irreversible action attempted without verified authority).
- **Decision:** `HOLD`.
- **Outcome:** The system MUST pause and require explicit human approval before any physical manifestation of the intent occurs.

## 7. Authority Model
- The **Machine** MUST be structured strictly as a metabolic processor. It has zero intrinsic authority.
- The **Sovereign (Human)** holds absolute veto capability. If an action's reversibility is zero, the Machine's authority converges to zero.

## 8. Execution Model
The prescribed architectural pattern is **Sense → Reason → Critique → Judge → Seal**.
Outputs MUST be generated in environments where internal thoughts are securely separated from the execution module.

## 9. Compliance Requirements
Any system claiming "arifOS Compliance" MUST:
1. Provide a verifiable implementation of all 13 Floors.
2. Implement a fail-closed response to state failures.
3. Be capable of generating Cryptographic Ledger proofs (Seals) for historical execution audits.

## 10. Conformance Levels
- **Level 1 (Logical Conformance):** System MUST structurally separate reasoning and expose explicit decision states (SEAL / HOLD / VOID / SABAR) externally.
- **Level 2 (Thermodynamic Conformance):** System MUST explicitly log execution metrics (e.g., $\Delta S$, $\Omega_0$) per execution cycle.
- **Level 3 (Sovereign Conformance):** System MUST log all SEAL boundaries in a Cryptographically Identity-bound and immutable Vault Ledger.

## 11. Versioning Policy
This document adheres to Semantic Versioning. Major versions represent structural shifts to the floors or states. Minor versions adjust thresholds.

## 12. Non-Goals
To prevent misuse and misinterpretation, this standard:
- Does **NOT** define model architecture (it is strictly LLM-agnostic).
- Does **NOT** require a specific programming language for implementation.
- Does **NOT** guarantee absolute correctness of generated outputs, only the safety and containment of their intent.
