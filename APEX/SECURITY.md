## Reporting a Vulnerability

**DO NOT** create a public GitHub issue for security vulnerabilities.

If you discover a security vulnerability within arifOS, please report it privately to the Sovereign (Arif) via the secure channels defined in the [SOUL.md](https://github.com/ariffazil/ariffazil/blob/main/SOUL.md).

Alternatively, you may send an encrypted email with the subject `[VULN] arifOS Hardening Report` to the contact address in the [Identity Manifest](https://arifos.arif-fazil.com/.well-known/arifos.json).

### Evaluation Process

1. **Acknowledgment**: You will receive a receipt within 72 hours (Phoenix-72 cycle).
2. **Analysis**: We will verify the vulnerability and determine its impact on the **13 Constitutional Floors**.
3. **Fixing**: We will aim to fix the vulnerability in a private branch before merging to main.
4. **Disclosure**: Once fixed, we will publish a security advisory if the impact is critical.
