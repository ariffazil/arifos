## Horizon FastMCP Build Fix — WELD-006

### Problem
arifOS horizon server build failing pre-flight (`fastmcp inspect`) because discovery 
routes (`GET /robots.txt`, `/llms.txt`, `/wells.json`, `/.well-known/agent.json`, 
`/.well-known/arifos.json`) were added as `@app.get()` at module level in `server.py`. 
`app = mcp.http_app()` returns `StarletteWithLifespan` which doesn't support 
raw `.get()` decorators — fails during Cloud build inspect.

### Fix (commit 19e0e362)
1. **Removed** `@app.get()` routes from `server.py` (lines 419-437)
2. **Added** same routes to `arifosmcp/runtime/rest_routes.py` using existing `@route()` 
   decorator (wraps `mcp.custom_route()` — works during inspect)
3. **Added** `FileResponse` to `rest_routes.py` imports
4. **Fixed** `NameError` guard: `register_rest_routes` at line 408 referenced 
   `HARDENED_HANDLERS` which is only in try-block scope; added `if "HARDENED_HANDLERS" in globals()`
5. **VAULT999** sealed: `LGR-HORIZON-WELD006-FIX` / `GIT-19e0e362`
